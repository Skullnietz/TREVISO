Descripción del sistema

Espacio Treviso es un sistema web de administración de archivos XML fiscales (CFDI) del SAT mexicano. Permite a una firma contable gestionar facturas electrónicas de sus clientes, controlar depósitos, reembolsos, complementos de pago, notas de crédito y generar reportes contables.

El sistema opera con dos roles principales: Administrador (root) y Colaborador (empleado), cada uno con su propio panel y nivel de acceso.

Problemas de seguridad identificados

CRITICO: Los siguientes problemas deben ser la máxima prioridad en la reingeniería.
IDSeveridadProblemaUbicaciónRemediación
SEC-01 CRÍTICO SQL Injection — Todas las queries usan interpolación directa de variables $_POST/$_GET sin prepared statements Todo el proyecto (transaction*.php, vistas) Migrar a prepared statements (MySQLi o PDO)
SEC-02 CRÍTICO Contraseñas en texto plano en BD y validación directa con string transactionValLogin.php, usuarioempleado Implementar password_hash() / password_verify()
SEC-03 CRÍTICO Credenciales hardcodeadas de BD visibles en código fuente bd.php (3 copias), backup.php Mover a archivo .env o config fuera del webroot
SEC-04 ALTO Sin protección CSRF en la mayoría de formularios. Solo run_sync.php tiene token Todos los forms POST Implementar tokens CSRF globalmente
SEC-05 ALTO XSS potencial — Datos de BD mostrados sin htmlspecialchars() en múltiples vistas Vistas colaborador y root Sanitizar todas las salidas HTML
SEC-06 MEDIO backup.php expone estructura completa de la BD sin restricciones adicionales root/backup.php Agregar 2FA o confirmación para backups
SEC-07 MEDIO Sesión no regenerada tras login (session fixation) transactionValLogin.php Usar session_regenerate_id(true) tras login

Deuda técnica

IDCategoríaDescripciónImpacto
TD-01 Duplicación Los paneles root/ y colaborador/ comparten ~80% del código con variantes mínimas (filtro por IDEmpleado y menú reducido) Mantenimiento doble; bugs se arreglan en un panel y no en otro
TD-02 Duplicación Existen 3 copias de bd.php (raíz, root/transactions, colaborador/transactions) con las mismas credenciales Inconsistencia potencial al cambiar credenciales
TD-03 Arquitectura Sin separación MVC real: las vistas contienen queries SQL directas Imposible hacer testing unitario; lógica acoplada
TD-04 Librería PHPExcel está deprecada (2017). El reemplazo oficial es PhpSpreadsheet Vulnerabilidades no parcheadas; incompatibilidad con PHP 8+
TD-05 Frontend jQuery 2.x sin framework moderno. CSS inline mezclado con archivos. Sin minificación. Performance pobre; difícil de mantener
TD-06 Naming Archivos de backup con fechas en el nombre (ej. headers_backup 18-11-2022.php) Archivos muertos en producción; confusión
TD-07 Infraestructura Dependencia de rutas Windows absolutas (E:\XML\, C:\TrevisoLogs\) Imposible migrar a Linux/containers sin refactor
TD-08 Sesiones Slides almacenados en $_SESSION (datos de UI en sesión del servidor) Uso innecesario de memoria; problemas de concurrencia

Recomendaciones para reingeniería

Fase 1: Seguridad (inmediata)

  • Migrar todas las queries a prepared statements (PDO recomendado)
  • Implementar password_hash/verify para contraseñas
  • Mover credenciales a un archivo .env fuera del webroot
  • Agregar protección CSRF global
  • Sanitizar toda salida HTML con htmlspecialchars
  • Regenerar session ID tras login exitoso

Fase 2: Consolidación de código

  • Unificar root/ y colaborador/ en una sola base con sistema de permisos
  • Crear un único bd.php con clase de conexión centralizada
  • Extraer la lógica de negocio de las vistas a clases de servicio
  • Eliminar archivos backup/muertos del proyecto
  • Reemplazar PHPExcel por PhpSpreadsheet

Fase 3: Arquitectura moderna

  • Adoptar un framework PHP (Laravel, Symfony) o al menos un micro-framework (Slim)
  • Implementar patrón MVC con rutas, controladores y modelos
  • Migrar frontend a un framework moderno (Vue.js, React) con API REST
  • Implementar sistema de migraciones para la BD
  • Containerizar con Docker para eliminar dependencia de rutas Windows
  • Agregar testing automatizado (PHPUnit, Cypress)
